¿Qué ocurre cuando ejecutas node server.js?
Node.js lee el archivo server.js, carga Express y pone el servidor en funcionamiento,
quedando activo y esperando solicitudes entrantes en el puerto 3000.

¿Qué mensaje aparece en la terminal?
Aparece el siguiente mensaje:
"Servidor ejecutándose en puerto 3000"
Este mensaje lo genera el callback dentro de app.listen cuando el servidor
inicia correctamente.

¿Qué sucede cuando visitas la ruta /estado?
El servidor responde con un objeto JSON que confirma que está funcionando:
{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}
Esta ruta es útil para verificar rápidamente si el servidor está activo
y respondiendo correctamente a las solicitudes.
